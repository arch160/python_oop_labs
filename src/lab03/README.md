# Лабораторная работа №3

## Цель
Сделать наследников от уже существующего класса, добавлять им новые поля и методы, переопределять поведение и использовать полиморфизм.

## Что получилось

Базовый класс — `Book` из первой лабы. От него сделано два дочерних: `Ebook` и `AudioBook`.

### `Ebook` (электронная книга)
Есть:
- размер файла в МБ
- флаг защиты DRM (есть или нет)

Новый метод: `download()` — возвращает строку про скачивание.

Переопределил `__str__`, чтобы в выводе добавлялась информация про формат и защиту.

### `AudioBook` (аудиокнига)
Есть:
- длительность в часах
- имя чтеца (нарратор)

Новый метод: `listen_preview()` — имитация прослушивания отрывка.

`__str__` переопределён — показывает длительность и чтеца.

### Полиморфизм
В обоих наследниках метод `process()`. У `Ebook` он вызывает `download()`, у `AudioBook` — `listen_preview()`. Это сделано, чтобы можно было вызвать один и тот же метод у разных объектов и получить разное поведение, не проверяя тип.

## Коллекция из ЛР2
Использую тот же `Library` из второй работы. В нём можно хранить и обычные книги, и электронные, и аудиокниги — потому что все они наследники `Book`.

**Сценарий 1 - просто наследование**
Созданы объекты всех трёх типов, вывожу их (смотрю, как работают переопределённые `__str__`), вызываю `download()` и `listen_preview()`. Смотрим, что у электронной книги тоже работает метод выдачи из базового класса.

**Сценарий 2 - полиморфизм и коллекция**
Кладем в библиотеку разные книги. Потом в цикле проверяю, есть ли у объекта метод `process()` (через `hasattr`), и вызываю его. Для электронной книги - одно сообщение, для аудиокниги - другое, обычная книга ничего не выводит. 

**Сценарий 3 - фильтрация по типу**
Есть функция, которая фильтрует коллекцию и оставляет только объекты указанного класса. Выводим отдельно электронные книги и отдельно аудиокниги.
